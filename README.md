# Репетиции — планировщик постановки

Веб-приложение для планирования театральных репетиций.

## Возможности

- **Актёры** — состав труппы с фото, ролями и контактами
- **Пьеса** — информация о постановке
- **Сцены** — список сцен для прохождения со статусами
- **Задачи** — задачи для репетиций с назначением актёров
- **Календарь репетиций** — планирование по дням (заготовка под Google Календарь)
- **План по времени** — почасовой план репетиции: сцены, задачи, перерывы

## Хранение данных

Основное хранилище — **SQLite** на сервере (`data/rehearsals.db`). Фронтенд работает через API (`/api/state`); без запущенного API приложение не загружается.

Дополнительно в **localStorage** браузера дублируется состояние (резерв при восстановлении) и сохраняются настройки интерфейса (режим оформления, свёрнутая боковая панель).

Резервные копии SQLite — в `data/backups/`.

## Запуск

```bash
npm install
npm run dev
```

Или на Windows: `restart.bat`

При локальном запуске откройте **http://localhost:3003** (API — http://localhost:3001).

Скопируйте `.env.example` в `.env` и заполните при необходимости (Telegram, Google OAuth).

## Сборка

```bash
npm run build
```

Запуск API в production (порт из `API_PORT`, по умолчанию 3001):

```bash
npm start
```

На сервере: `pm2 start npm --name rehearsals-api -- start`

Пример конфига nginx (статика из `dist/`, `/api` → `:3001`): [`deploy/nginx.conf.example`](deploy/nginx.conf.example)
